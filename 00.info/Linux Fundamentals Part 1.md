## Задание 2 Немного предыстории Linux
#### Где используется Linux ?
Справедливо сказать, что подход к Linux гораздо более устрашающий, чем к операционным системам (ОС), таким как 
Windows. Оба варианта имеют свои преимущества и недостатки. Например, Linux значительно легче, и вы будете удивлены,
узнав, что есть большая вероятность, что вы использовали Linux в той или иной форме каждый день! Linux поддерживает 
такие вещи, как:

Веб-сайты, которые вы посещаете
- Автомобильные развлечения/панели управления
- Системы точек продаж (PoS), такие как кассы и кассы в магазинах.
- Критически важные инфраструктуры, такие как контроллеры светофоров или промышленные датчики.

#### Особенности Linux
Название «Linux» на самом деле является общим термином для нескольких ОС, основанных на UNIX (другой операционной 
системе). Благодаря тому, что Linux имеет открытый исходный код, варианты Linux доступны во всех формах и размерах, 
лучше всего подходящих для того, для чего используется система.

Например, Ubuntu и Debian являются одними из наиболее распространенных дистрибутивов Linux, поскольку они очень 
расширяемы. Т.е. вы можете запускать Ubuntu как сервер (например, веб-сайты и веб-приложения) или как полноценный 
рабочий стол. В этой серии мы будем использовать Ubuntu.

Ubuntu Server может работать на системах с 512 МБ ОЗУ.


Подобно тому, как у вас есть разные версии Windows (7, 8 и 10), существует множество разных версий/дистрибутивов Linux .

Взаимодействие с файловой системой
Как я уже говорил ранее, очень важно иметь возможность перемещаться по машине, на которой вы вошли, не полагаясь на

Команда	Полное имя
- ls	- listing (листинг)
- cd	- change directory (изменить каталог)
- cat	- concatenate (объединять)
- pwd	- print working directory (распечатать рабочий каталог)

Вывод списка файлов в нашем текущем каталоге (ls)

очередь знать, что существует. Это можно сделать с помощью команды «ls» (сокращение от «листинг»)

и имя каталога. Т.е. `ls Pictures`


#### Изменение нашего текущего каталога (cd)
Теперь, когда мы знаем, какие папки существуют, нам нужно использовать команду « cd » (сокращение от «сменить

Pictures ». Опять же, мы хотим узнать содержимое этого каталога «Изображения» и для этого снова используем « ls »:

```commandline

dog_picture1.jpg dog_picture2.jpg dog_picture3.jpg dog_picture4.jpg
```
В данном случае, похоже, есть 4 изображения собак!

## Задание 6 Поиск файлов
Хотя пока это не так, но одна из положительных особенностей Linux — это то, насколько эффективно вы можете с ним 
работать. С учетом вышесказанного, вы, конечно, можете быть настолько эффективными, насколько вы с этим знакомы. По

Один из фантастических способов показать, насколько эффективно вы можете работать с такими системами, — это 
использовать набор команд для быстрого поиска файлов во всей системе, к которым у нашего пользователя есть доступ. 
Не нужно последовательно пользоваться  cd и ls выяснять, что где. Вместо этого мы можем использовать такие команды,
чтобы  find автоматизировать такие вещи для нас!

Здесь подход к Linux становится немного более пугающим, но мы разберемся в этом и облегчим вам задачу.

#### Использование поиска
Команда `find` является фантастической в том смысле, что ее можно использовать как очень просто, так и довольно 
сложно, в зависимости от того, что именно вы хотите сделать. Однако давайте сначала остановимся на основах.

Возьмите фрагмент ниже; мы можем увидеть список доступных нам каталогов:

Использование «ls» для отображения содержимого текущего каталога
```
tryhackme@linux1:~$ ls
Desktop Documents Pictures folder1
tryhackme@linux1:~$
```
Теперь, конечно, каталоги могут содержать в себе еще больше каталогов. Это становится головной болью, когда нам 
приходится просматривать каждый файл только для того, чтобы попытаться найти определенные файлы. 

Давайте начнем с простого и предположим, что мы уже знаем имя искомого файла, но не можем вспомнить, где именно он 
находится! В данном случае мы ищем «passwords.txt».

Если мы запомним имя файла, мы можем просто использовать `find -name passwords.txt` команду, которая будет 
просматривать каждую папку в нашем текущем каталоге в поисках этого конкретного файла, например:

Использование «найти» для поиска файла с именем «passwords.txt»
```
tryhackme@linux1:~$ find -name passwords.txt
./folder1/passwords.txt
tryhackme@linux1:~$
```
удалось найти файл — оказывается, он находится в папке1/passwords.txt — сладко. Но предположим, что мы не знаем 
имени файла или хотим найти каждый файл с расширением, например «.txt», давайте и мы сделаем это!

Мы можем просто использовать так называемый подстановочный знак (*) для поиска всего, что имеет .txt в конце. В 
нашем случае мы хотим найти каждый файл .txt в нашем текущем каталоге. Мы создадим такую команду, как find -name *.
txt. Где «Найти» смог найти каждый файл .txt и затем указал нам местоположение каждого из них:

Использование «найти» для поиска любого файла с расширением «.txt»
tryhackme@linux1:~$ find -name *.txt
./folder1/passwords.txt
./Documents/todo.txt
tryhackme@linux1:~$

Найти удалось:
- «passwords.txt», расположенный в папке «folder1».
- «todo.txt» находится в разделе «Документы».
Это было не так уж сложно, да!


#### Использование Грепа
Еще одна замечательная утилита, о которой стоит узнать, — это использование `grep`. Команда `grep` позволяет нам искать 
в содержимом файлa конкретные значения, которые мы ищем.

Возьмем, к примеру, журнал доступа веб-сервера. В этом случае в журнале доступа веб-сервера имеется 244 записи.

Использование «wc» для подсчета количества записей в «access.log»
```
tryhackme@linux1:~$ wc -l access.log
244 access.log
tryhackme@linux1:~$
```

Использование команды типа `cat` здесь не поможет. Скажем, например, если бы мы хотели выполнить поиск в этом файле 
журнала, чтобы увидеть, что посетил определенный пользователь/IP-адрес? Просмотр 244 записей не так уж и эффективен,
учитывая, что мы хотим найти конкретное значение.

Мы можем использовать grep для поиска по всему содержимому этого файла любые записи значения, которое мы ищем. На 
примере журнала доступа веб-сервера мы хотим увидеть все, что посетил IP-адрес «81.143.211.90» (обратите внимание, 
что это вымышленное значение).

Использование «grep» для поиска любых записей с IP-адресом «81.143.211.90» в «access.log».
```
tryhackme@linux1:~$ grep "81.143.211.90" access.log
81.143.211.90 - - [25/Mar/2021:11:17 + 0000] "GET / HTTP/1.1" 200 417 "-" "Mozilla/5.0 (Linux; Android 7.0; Moto G(4))"
tryhackme@linux1:~$
```
«Grep» просмотрел этот файл и показал нам все записи о том, что мы предоставили и содержится в этом файле журнала 
для IP-адреса.

## Задание 7 Введение в операторы оболочки
Операторы Linux — это фантастический способ расширить свои знания о работе с Linux. Есть несколько важных 
операторов, на которые стоит обратить внимание. Мы рассмотрим основы и разобьем их на небольшие кусочки.

В обзоре я собираюсь продемонстрировать следующие операторы:

Символ / Оператор	Описание
- `&`	Этот оператор позволяет вам запускать команды в фоновом режиме вашего терминала.
- `&&`	Этот оператор позволяет вам объединить несколько команд в одной строке вашего терминала.
- `>`	Этот оператор является перенаправителем — это означает, что мы можем взять вывод команды (например, используя 
  cat для вывода файла) и направить его куда-нибудь еще.
- `>>` Этот оператор выполняет ту же функцию, что и `>` оператор, но добавляет вывод, а не заменяет (то есть ничего не 
  перезаписывается).

Давайте рассмотрим их немного подробнее.


#### Оператор «&»
Этот оператор позволяет нам выполнять команды в фоновом режиме. Например, предположим, что мы хотим скопировать 
большой файл. Очевидно, это займет довольно много времени и лишит нас возможности сделать что-либо еще, пока файл 
не будет успешно скопирован.

Оператор оболочки «&» позволяет нам выполнить команду и запустить ее в фоновом режиме (например, эта копия файла), 
что позволяет нам делать другие вещи!



#### Оператор «&&»
Этот оператор оболочки немного вводит в заблуждение с точки зрения того, насколько знаком его партнер «&». В 
отличие от оператора «&», мы можем использовать «&&», чтобы составить список команд для запуска, например command1 
&& command2. Однако стоит отметить, что command2он будет работать только в случае command1успеха.



### Оператор «>»
Этот оператор известен как перенаправитель вывода. По сути, это означает, что мы берем выходные данные команды, 
которую запускаем, и отправляем их куда-то еще.

Отличным примером этого является перенаправление вывода команды echo, которую мы изучили в задаче 4. Конечно, 
запуск чего-то вроде echo howdy возврата «привет» обратно на наш терминал — это не очень полезно. Вместо этого мы 
можем перенаправить «привет» на что-нибудь, например, на новый файл!

Допустим, мы хотим создать файл с именем «добро пожаловать» и сообщением «привет». Мы можем запустить echo hey > 
welcomeтам, где мы хотим, файл, созданный с содержимым «эй», вот так:

Использование оператора `>`
```commandline
tryhackme@linux1:~$ echo hey > welcome
Использование cat для вывода файла «приветствия»
tryhackme@linux1:~$ cat welcome
hey
```
Примечание. Если файл «Добро пожаловать» уже существует, его содержимое будет перезаписано!


#### Оператор ">>"
Этот оператор также является перенаправителем вывода, как и предыдущий оператор ( >), который мы обсуждали. Однако 
отличие этого оператора заключается в том, что вместо того, чтобы, например, перезаписывать какое-либо содержимое в 
файле, он просто помещает выходные данные в конец.

Продолжая наш предыдущий пример, где у нас есть файл «приветствие» с содержимым «эй». Если бы мы использовали echo 
для добавления «привет» к файлу с помощью оператора > , в файле теперь будет только «привет», а не «привет».

Оператор `>>` позволяет добавлять вывод в конец файла, а не заменять содержимое следующим образом:

Использование оператора >>
```
tryhackme@linux1:~$ echo hello >> welcome
Использование cat для вывода файла «приветствия»
tryhackme@linux1:~$ cat welcome
hey
hello
```
