## Задание 4
Файловая система, используемая в современных версиях  Windows,  — это New Technology File System или просто  NTFS .

До NTFS существовали FAT16/FAT32 (таблица размещения файлов) и HPFS (высокопроизводительная файловая система). 

Разделы FAT используются и сегодня. Например, разделы FAT обычно встречаются на USB-устройствах, картах MicroSD и т. 
д.,  но традиционно не на персональных компьютерах/ноутбуках Windows или серверах Windows. 

NTFS известна как журналируемая файловая система. В случае сбоя файловая система может автоматически восстанавливать 
папки/файлы на диске, используя информацию, хранящуюся в файле журнала. Эта функция невозможна в FAT.    

NTFS устраняет многие ограничения предыдущих файловых систем, такие как: 

Поддерживает файлы размером более 4 ГБ
Установите определенные разрешения для папок и файлов
Сжатие папок и файлов
Шифрование ( файловая система шифрования или EFS )
Если вы используете Windows, какую файловую систему использует ваша установка Windows? Вы можете проверить свойства 
(щелчок правой кнопкой мыши) диска, на котором установлена ваша операционная система, обычно это диск C (C:\). 

 

Официальную документацию Microsoft по FAT, HPFS и NTFS можно прочитать здесь .

Как просмотреть разрешения для файла или папки?

Еще одной особенностью NTFS являются альтернативные потоки данных ( ADS ).

Альтернативные потоки данных  (ADS) — это атрибут файла, специфичный для Windows  NTFS  (New Technology File System).

Каждый файл имеет по крайней мере один поток данных ( $DATA), а ADS позволяет файлам содержать более одного потока 
данных. Изначально Window Explorer не отображает ADS для пользователя. Существуют сторонние исполняемые файлы, 
которые можно использовать для просмотра этих данных, но Powershell дает вам возможность просматривать ADS для файлов.

С точки зрения безопасности создатели вредоносных программ используют ADS для сокрытия данных.

Не все его применения вредоносны. Например, когда вы загружаете файл из Интернета, в ADS записываются идентификаторы,
указывающие, что файл был загружен из Интернета. 

Чтобы узнать больше об ADS, перейдите по следующей ссылке от MalwareBytes здесь.

## Задание 6
В типичной локальной системе Windows учетные записи пользователей могут быть двух типов: администратор и  стандартный пользователь . 

Тип учетной записи пользователя определяет, какие действия пользователь может выполнять в данной конкретной системе Windows. 

Администратор может вносить изменения в систему: добавлять пользователей, удалять пользователей, изменять группы, изменять настройки системы и т. д. 
Стандартный пользователь может вносить изменения только в папки/файлы, атрибутированные пользователю, и не может выполнять изменения на уровне системы, такие как установка программ.
В данный момент вы вошли в систему как Администратор. Существует несколько способов определить, какие учетные записи пользователей существуют в системе. 

Один из способов — нажать Start Menuи ввести Other User. Должен появиться ярлык  System Settings > Other users. 



Если вы нажмете на нее, то должно появиться окно настроек. Смотрите ниже.



Поскольку вы являетесь администратором, вы видите опцию Добавить кого-то еще к этому ПК .

Примечание : Обычный пользователь не увидит эту опцию.  

Нажмите на локальную учетную запись пользователя. Должны появиться дополнительные параметры: Изменить тип учетной записи и Удалить . 



Нажмите Изменить тип учетной записи. Значение в раскрывающемся списке (или выделенное значение, если вы нажмете на раскрывающийся список) — это текущий тип учетной записи. 



При создании учетной записи пользователя создается профиль для пользователя. Расположение каждой папки профиля пользователя будет C:\Users.

Например, папка профиля пользователя для учетной записи Макс будет C:\Users\Max.

Создание профиля пользователя выполняется при первом входе в систему. Когда новый пользователь впервые входит в локальную систему, он увидит несколько сообщений на экране входа. Одно из сообщений, User Profile Service, находится на экране входа в систему некоторое время, создавая профиль пользователя. Смотрите ниже.

 

После входа в систему пользователь снова увидит диалоговое окно, похожее на показанное ниже, указывающее на то, что профиль находится в процессе создания.

Рабочий стол

Подавляющее большинство домашних пользователей входят в свои системы Windows как локальные администраторы. Помните из предыдущей задачи, что любой пользователь с типом учетной записи администратор может вносить изменения в систему.

Пользователю не нужно работать с высокими (повышенными) привилегиями в системе для выполнения задач, не требующих таких привилегий, таких как серфинг в Интернете, работа с документом Word и т. д. Эта повышенная привилегия увеличивает риск компрометации системы, поскольку она облегчает заражение системы вредоносным ПО. Следовательно, поскольку учетная запись пользователя может вносить изменения в систему, вредоносное ПО будет работать в контексте вошедшего в систему пользователя.

Для защиты локального пользователя с такими привилегиями Microsoft ввела Контроль учетных записей (UAC). Эта концепция была впервые представлена ​​в недолговечной Windows Vista  и продолжилась в последующих версиях Windows.

Примечание : UAC (по умолчанию) не применяется к встроенной учетной записи локального администратора. 

Как работает UAC? ​​Когда пользователь с типом учетной записи администратора входит в систему, текущий сеанс не запускается с повышенными разрешениями. Когда необходимо выполнить операцию, требующую более высоких привилегий, пользователю будет предложено подтвердить, разрешает ли он выполнение операции. 

Давайте рассмотрим программу в учетной записи, в которую вы в данный момент вошли, встроенной учетной записи администратора. Щелкните правой кнопкой мыши, чтобы просмотреть ее свойства.

На вкладке Безопасность мы можем увидеть пользователей/группы и их разрешения на этот файл. Обратите внимание, что стандартный пользователь не указан. 



Войдите в систему как стандартный пользователь и попробуйте установить эту программу. Для этого вы можете подключиться к удаленному рабочему столу машины как стандартная учетная запись пользователя. 

Примечание : У вас есть имя пользователя и пароль для стандартного пользователя. Они видны в lusrmgr.msc.

Перед установкой программы обратите внимание на значок. Видите разницу?  Когда вы вошли в систему как обычный пользователь, значок щита находится на значке программы по умолчанию. Смотрите ниже.



Этот значок щита является индикатором того, что UAC запросит разрешение на установку программы с более высокими привилегиями.



Дважды щелкните программу, и вы увидите запрос UAC. Обратите внимание, что встроенная учетная запись администратора уже установлена ​​как имя пользователя и запрашивает пароль учетной записи. Смотрите ниже.



Если пароль не введен, через некоторое время запрос UAC исчезает, и программа не устанавливается. 

Эта функция снижает вероятность того, что вредоносное ПО успешно скомпрометирует вашу систему. Подробнее об UAC можно прочитать здесь.
